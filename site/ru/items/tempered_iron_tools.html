Инструменты из закалённого железа

alaindustrial:tempered_iron_tools
Игровые предметыРеализованоtooltempered-irontier-lv

Сводка

Инструменты из закалённого железа — линейка из пяти ручных инструментов мода Ala Industrial: кирка, топор, мотыга, лопата и меч. Каждый крафтится как обычный ванильный инструмент (vanilla-pattern верстака), но вместо iron_ingot используется закалённое железо мода. По всем характеристикам немного лучше ванільных железных аналогов (прочность, скорость, урон, зачарование), но инструменты добычи (кирка/топор/лопата/мотыга) не майнят diamond-level блоки — материал переиспользует minecraft:incorrect_for_iron_tool, поэтому алмазная прогрессия не обесценивается. Все пять инструментов используют единый материал ModToolMaterials.TEMPERED_IRON (record ToolMaterial MC 26.2). В MC 26.2 классы PickaxeItem/SwordItem/DiggerItem/Tier/Tiers удалены — кирка и меч это обычные Item с data-компонентом Tool через Item.Properties.{pickaxe,sword}(ToolMaterial, attackDamage, attackSpeed). Однако AxeItem, HoeItem, ShovelItem всё ещё существуют и несут signature-поведение (useOn: снятие коры, вспашка, тропинка) — поэтому топор/мотыга/лопата наследуются от этих ванильных подклассов; их конструкторы сами вызывают props.{axe,hoe,shovel}(...) и super(...), обеспечивая и Tool-компонент, и поведение.

Характеристики (баланс)

Базовая линия — ванильные железные инструменты. Значения аргументов attackDamage/attackSpeed взяты 1:1 из байткода Items.IRON_* (проверено javap против minecraft-common-deobf-26.2.jar); единственное улучшение — в материале ModToolMaterials.TEMPERED_IRONattackDamageBonus = 2.5 (vs iron 2.0), durability = 350 (vs 250), speed = 7.0 (vs 6.0), enchantmentValue = 16 (vs 14). Это даёт +0.5 итогового урона и +100 прочности каждому инструменту автоматически — тот же подход, что у кирки. Формула урона в 26.2 (из ToolMaterial.createToolAttributes): итоговый ATTACK_DAMAGE игрока = 1.0 (base) + (attackDamage + attackDamageBonus). ATTACK_SPEED = 4.0 (base) + attackSpeed.

Любой из пяти инструментов чинится в наковальне 1 закалённым железом.

Поведение

Добыча (кирка/топор/лопата/мотыга): копают блоки соответствующего ванильного тега (mineable/pickaxe / mineable/axe / mineable/shovel / mineable/hoe) со скоростью материала 7.0. Diamond-level блоки (тег minecraft:incorrect_for_iron_tool) роняются без дропа — идентично железным инструментам. Топор: дополнительно — снятие коры (stripping), воспламенение (правый клик по дереву), разрушение деревянных блоков быстрее. Поведение ванильного топора через AxeItem.useOn (подкласс сохранён в 26.2, в отличие от PickaxeItem/SwordItem). Мотыга: вспашка земли, сбор урожая, ванильные способности hoe через HoeItem.useOn. Лопата: формирование тропинки (path), тушение костра через ShovelItem.useOn. Меч: урон 6.5 (vs железо 6.0), мгновенная добыча паутины, отбрасывание (Weapon-компонент). Боёвка: урон каждого на +0.5 выше железного аналога (см. таблицу выше), скорость атаки идентична железу. Не требует EU/FE — чисто ванильная механика износа. Прочность: 350 (на 100 больше железа). Расходуется за добычу и удары. Зачарование: enchantmentValue 16 (железо 14) — материал «чуть лучше ловит чары». Инструменты добавлены в membership-теги #minecraft:{pickaxes,axes,hoes,shovels,swords}, поэтому через цепочку enchantable-тегов (#minecraft:enchantable/{mining,weapon,durability}) зачаровывающий стол предлагает полный набор ванильных чар — см. Совместимость. Ремонт: в наковальне закалённым железом через общий ремонт-тег.

Прогрессия

Тир: none (ручные инструменты, не привязаны к EnergyTier мода). Занимают слот между ванильными железными и алмазными инструментами: по прочности/скорости ближе к железу, не перекрывают алмаз по доступу к блокам. Появляются на Фазе 1, как только игрок начинает массово переплавлять железо в закалённое — естественный апгрейд до первой алмазной экипировки.

Совместимость

Просмотрщики рецептов: JEI, REI, EMI — как обычные предметы с крафтом. Зачарования: проектные — все ванильные (efficiency, unbreaking, fortune, silk_touch, sharpness, mending и т.д.) как стандартные наборы для каждого типа инструмента. Инструменты добавлены в membership-теги #minecraft:{pickaxes,axes,hoes,shovels,swords} (5 JSON data/minecraft/tags/item/*.json, replace: false — , по аналогии с бронёй ). По цепочке enchantable-тегов (#minecraft:enchantable/{mining,weapon,durability}) зачаровывающий стол предлагает полный набор: кирка/топор/лопата/мотыга — efficiency/unbreaking/fortune/silk_touch/mending; меч — sharpness/looting/fire_aspect/unbreaking/mending. Advancement minecraft:story/iron_tools срабатывает на крафте кирки. Причина, по которой теги нужны отдельным JSON: Item.Properties.{pickaxe,axe,hoe,shovel,sword}(...) в MC 26.2 ставят только Tool-компонент майнинга и не регистрируют предмет в теге (подтверждено javap -c); ENCHANTABLE-компонент выставляется через ToolMaterial.applyCommonProperties → Properties.enchantable(enchantmentValue), но фильтр стола идёт по supported_items → тегам, не по компоненту. Ремонт в наковальне работает через tempered_iron_tool_materials независимо от membership-тегов.
