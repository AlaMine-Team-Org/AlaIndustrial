Железный сундук

alaindustrial:iron_chest
Утилитарные блокиРеализованоstoragechestutility

Сводка

Железный сундук — чистое хранилище предметов на 36 слотов (4 ряда × 9), на один ряд больше ванильного 27-слотового сундука. Не связан с энергосетью мода: это «гражданский» блок хранения, апгрейд ванильного медного сундука (minecraft:copper_chest из Copper Age drop 26.2). Крафтится медным сундуком в центре, окружённым 8 железными слитками. Рендерится как настоящая 3D-модель сундука с анимированной крышкой (как ванильный деревянный сундук) и собственной текстурой iron.png (64×64, тот же UV-разворот). При открытии/закрытии играет фирменный звук iron_chest_open/iron_chest_close. В отличие от машин мода, BE построен не на MachineBlockEntity (который требует энергию), а на ванильном — это первый контент-блок мода без энергозависимости.

Интерфейс

Тип: container (, alaindustrial:iron_chest). GUI-атлас: assets/alaindustrial/textures/gui/container/iron_chest.png (256×256, видимая часть 176×184 — выше стандартного 166 из-за 4-го ряда). Без индикаторов энергии/прогресса (сундук не машина). Обе подписи слева: заголовок «Железный сундук» над сеткой сундука (titleLabelX = 8) и подпись «Инвентарь» в зазоре между сундуком и инвентарём игрока (inventoryLabelX = 8, inventoryLabelY = 92).

Рендер и анимация

3D-модель через на 26.2 ): переиспользует ванильную ChestModel (lid/lock/bottom, createSingleBodyLayer), bake из собственного alaindustrial:iron_chest. Текстура — assets/alaindustrial/textures/entity/chest/iron.png (64×64, формат ванильного entity/chest/normal). Подаётся через atlas-sprite path ( + ) — как ванильный .submit, что даёт корректный depth/cull/mipmaps (без белой обводки). Анимация крышки через ChestLidController: openness интерполируется клиентским тикером .; кривая 1 - (1-open)³ совпадает с ванилью. Трансформация по facing — (yaw вокруг центра-сверху). Item-иконка — 3D special renderer (items/iron_chest.jsontype: minecraft:special, model: {type: minecraft:chest, texture: alaindustrial:iron} + base: minecraft:item/chest). Маркетинговый 3D-рендер — tools/iron_chest_3d.png (128px, апскейлится до 512 в ассет-паке; закрытый сундук с крышкой, изометрия, прозрачный фон). Генерируется процедурно из модельной текстуры entity/chest/iron.png по ванильной chest-геометрии (ChestModel.createSingleBodyLayer — bottom 14×10×14 + lid 14×5×14 + lock 2×4×1) скриптом tools/gen_chest_3d.py; кубическая render_cube сюда не подходит — ванильная chest-геометрия плоскими гранями не выражается. Производный рендер tools/iron_chest_open_preview_128.png — то же в открытом состоянии (крышка поднята); лежит «на всякий случай», в HTML пока не используется. См. .

Звук

iron_chest_open / iron_chest_close — сгенерированы через ElevenLabs SFX (та же текстура-кандидат close_wood_02 использована для обоих), конвертированы MP3→OGG (mono, 44.1kHz, Vorbis). Проигрываются из .onOpen/onClose через playSound(...): источник SoundSource.BLOCKS, громкость 0.42 (тише ванильного 0.5 — звук сундука звонкий), случайный pitch 0.9–1.0. Субтитры: subtitles.alaindustrial.iron_chest_open / ..._close (полная локализация, 20 языков).

Рецепты

Крафт: data/alaindustrial/recipe/iron_chest.json Тип: minecraft:crafting_shaped, категория redstone. Pattern 3×3: III / ICI / III. C = #minecraft:copper_chests (любая стадия окисления + waxed), I = minecraft:iron_ingot. Результат: alaindustrial:iron_chest.

Совместимость

Просмотрщики рецептов: JEI, REI, EMI (через существующий AlaReiPlugin). Воронки: сундук реализует ), поэтому воронки/трубы могут вставлять/извлекать предметы стандартным ванильным путём (симметричный IO — без ролей слотов, т.к. все слоты одинаковые).

Прогрессия

Линейка сундуков: ванильный copper_chest (27) → Iron Chest (36) → (потенциально) следующие тиры (silver/gold/diamond — не описаны).

Связанное

Энергохранилище — другой «storage»-блок мода, но энергитический. Ванильный minecraft:copper_chest — предшественник в крафте. tools/tts/generate_sfx.py (приватный, не публикуется) — генератор SFX-кандидатов через ElevenLabs.
