Stock Display Frame

alaindustrial:stock_display_frame
UtilityLVImplementedstoragedisplayitem-frameutilitylogistics

Summary

The Stock Display Frame is a frame you hang on any container (a vanilla chest, barrel, or shulker box, or the mod's Iron Chest) — and it shows the item count inside it right on the frame, in real time. It uses no energy: it's a warehouse convenience — "price tags" for rows of chests, so you can read how full they are without opening a single window. How the number is counted: an item is in the frame → the number equals how many items of that type are in the container (the item acts as a filter); the frame is empty → no number is shown (the frame looks like a plain one); no container behind the frame → no number; the container is empty (with a filter in the frame) → shows 0. Only top-level slots are counted — it does not dig into shulkers or bundles inside the chest. A double chest counts as one container: a frame on either half shows the combined total.

Mechanics

Placing and removing

The frame is placed and removed like the vanilla one: right-click a block face to hang it, hit it to take it down. If you break the block behind it, the frame pops off and drops its own item. The frame item is placed with a right click on a block face (vanilla placement — like a plain frame, horizontal faces + top/bottom). The frame holds by the vanilla rules: break the supporting block and it pops off, dropping its own item (not a vanilla frame). Hitting a filled frame knocks the filter item out (first hit); a second hit removes the frame — inherited vanilla behaviour.

Interaction (right click)

Unlike the vanilla frame, the item in it does not rotate — that's just noise for a filter:

Emptyput the filter item in (1 pc) → the number appearsnothing
Filteredtake the filter back → the number disappearstake the filter back → the number disappears

Maps (minecraft:filled_map) cannot be placed in the frame — filtering maps in a chest is meaningless.

Counting the contents

Once per second the frame recounts the contents of the container behind it. A double chest counts as one; with a filter, only stacks of the same type as the frame's item are counted. Once a second (20 ticks by default) the frame scans the block behind it. If the needed chunk isn't loaded, the frame doesn't force-load it — it just skips that tick. The container is resolved like a vanilla comparator would: a double chest as one, a barrel and a shulker as ordinary containers, the mod's Iron Chest and third-party containers too. Item counts are summed across slots; with a filter, only stacks matching the filter item by type and components. If there's no container behind the frame, the count is "no container" and no number is shown.

Displaying the number

The number is drawn flat on the frame's face, in its bottom strip — white with a dark outline, no backdrop. It turns with the frame (not a billboard), so it's only visible from the side the frame faces. The filter item is raised and shrunk a little to make room for the digit; without a filter the frame looks like a plain one (item centred). Display distance — up to 24 blocks (the text hides beyond that, for performance). Formatting: the exact number below 10,000, abbreviated after that — 12.3k, 1.2M. An empty frame or "no container" renders no text at all.

Redstone

A comparator reading the frame gets the container fullness behind it — on the same 0–15 curve as a comparator directly on the chest. The signal ignores the filter: it's the full container's fullness. No container → signal 0.

Block properties

The frame is an entity, not a block (like the vanilla frame). It's the same size and behaves the same way: flat, on a wall, no collision.

Recipes

Crafted at a workbench: a vanilla item frame in the centre, surrounded by gears, silver and gold dust. The idea is to "upgrade" an ordinary frame into a counting one with precise mechanics and precious electronics. The recipe unlocks in the recipe book once you have a gear or silver. Result: 1 × alaindustrial:stock_display_frame.

Compatibility

Works with any ordinary container: vanilla chest / trapped chest / copper chest / barrel / shulker box, the mod's Iron Chest, and third-party containers. Does not count inventories exposed only through third-party item-transfer APIs (a deliberate v1 simplification). Hoppers and pipes don't interact with the frame — it's an entity, not a block.

Progression

LV tier: available after the first gears and silver (recipe: a vanilla item frame + 2 wooden gears + 2 silver ingots + 4 gold dust).

Related

Iron Chest — the frame's main in-mod customer. Vanilla minecraft:item_frame / minecraft:glow_item_frame — the mechanic's ancestor.
