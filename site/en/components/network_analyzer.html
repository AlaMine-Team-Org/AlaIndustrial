Network Analyzer

alaindustrial:network_analyzer
ComponentsImplementedcomponentdiagnosticsenergytier-lv

Summary

The Network Analyzer is a hand-held diagnostic tool — an alternative to the /ala net command for players without operator rights. Right-clicking any cable shows live stats for that specific network (not the whole level): the combined EU/FE/t supply of producers, the combined EU/FE/t demand of consumers, and how much EU/FE actually moved over the last tick. At the same time it highlights the network in the world — cables and the producers/consumers attached to them glow as coloured dots, neighbouring cables (and producers/consumers) are joined by lines across the whole topology, and animated dots run along the lines showing the direction of energy flow from producer to consumer — similar to the channel visualisation in Applied Energistics 2.

Behavior

Two diagnostic modes, stored on the tool itself (the data component alaindustrial:network_analyzer_mode → the AnalyzerMode enum, persistent + network-synchronized): "Traverse" (default). A Battery Box is treated as part of the network: the analyzer walks through the storage-sink and highlights the entire connected cable system — even if topologically it's several distinct networks (for example [Gen]─cable─[BatBox]─cable─[Machine], where the Battery Box is an endpoint, not a cable, and so doesn't "stitch" the networks at the NetworkManager.register level). The server runs a read-only BFS (NetworkTraverser): it starts at the clicked network, and for each storage-sink checks the 6 neighbours and crosses into the adjacent network, with cycle protection (a visited-set of networks) and a cap of Config.networkAnalyzerMaxTraversedNetworks. "Stop-at-storage". Shows exactly one network — the one the clicked cable is in; the walk stops at a storage-sink (the original behaviour). Handy for targeted diagnosis of a single segment. Switching modes: Shift + right-click in the air or off-network (including on someone else's block) cycles the mode 1↔2 and sends an actionbar message with the new mode's name. A plain right-click on a cable (Shift or not) scans in the current mode (Shift+cable doesn't switch — a click on a cable is always diagnostic). Right-click on a network's cable → the server finds the network via NetworkManager.networkAt and walks it (or several networks in traverse mode) through NetworkTraverser. It reads producerSupplyEstimate (total extractable EU/FE/t), consumerDemandEstimate (total acceptable EU/FE/t), lastTickMoved (actually moved over the last tick), plus the number of cables and the number of producer/consumer/storage endpoints. It prints an actionbar line with cables, sources/consumers/storages and supply/demand/last-tick (aggregated across all traversed networks in traverse mode), and sends the client a list of positions to highlight. When the network cap is reached in traverse mode, an extra actionbar warning is shown. Item tooltip (client, ItemTooltipCallback) — the base hint is always visible; below it, the current-mode line. The last scan is stored right on the tool (the data component alaindustrial:network_scan → the NetworkScanData record, persistent + networkSynchronized) and expands under [SHIFT]: cables, sources/consumers/storages, supply/demand EU/FE/t, transferred over the last tick. Right-click on another network's cable — replaces the highlight with the new network. Right-click off-network (air, a block without a network) — clears the highlight. The highlight is rendered in world coordinates through gizmo primitives, visible through walls and across several chunks; positions in client-unloaded chunks simply aren't drawn — there's no separate timer/TTL. The style is a "circuit-board trace": a volumetric square tube (hollow, 4 side faces, single-sided with outward normals) along the full network topology (cable-cable and cable-endpoint), with solid cubes at the joints (corners, forks, dead-ends, endpoints — green producer, orange consumer, light-blue storage-sink in traverse mode, to visually flag the "bridge node"). Animated flow pulses run along the tube from producer to consumer — short (0.1 block) warm-white segments of the tube itself: the walls are drawn as alternating sub-quads "dark/light", and the light segments move (1 pulse per block). The pulse is neither geometry laid over the top nor a pixel dot: vanilla point-gizmos size in screen pixels without perspective (a sub-pixel is unreadable), and any overlay above the wall loses the depth test at distance; repainting the wall is occluded by nothing and reads from any distance and angle. The direction on each line is set by the BFS distance from the nearest producer — on unpowered sections (a network with no producer, an isolated island) the tube is drawn statically, without animation. The render is the same on both loaders — the shared NetworkOverlayRenderer in common/, submitted per frame (Fabric — LevelRenderEvents, NeoForge — SubmitCustomGeometryEvent). "Wire-like" geometry (implementation detail, see NetworkTopology/NetworkOverlayRenderer): markers and the tube are anchored to the centre of the block cell on X/Z (not to the centre of the cable's VoxelShape, which is offset by the connection sleeves) — so the tube runs strictly along the cable centres and follows their routing with crisp right angles; on Y, endpoints sit on the block's real shape — for a half-slab (a solar panel, maxY = 0.5) the anchor is dropped below the top face by half the endpoint-cube's size, so the marker lies flush with the panel's top and doesn't protrude, and the incoming tube "descends" to the panel like the cable model's arm_low; straight runs between joints merge into a single seamless tube segment (tubeRuns) — with no internal seams per block; a joint cube is drawn only where the node is a real "corner/fork/dead-end" (jointNodes) — on straight tube blocks there's no joint, the run is seamless; an endpoint is always a joint (even pinched between two coaxial cables): the marker is drawn, the tube breaks at it and "descends" to the half-block from both sides. It doesn't change the network's state and doesn't affect energy distribution (it only reads).

Progression

Tier: none (a diagnostic tool, not tied to an EnergyTier). Useful from the first network onward, and increasingly so as networks grow more complex.

Compatibility

Recipe viewers: JEI, REI, EMI (as a regular item).

Balance

Not consumed and has no durability — a purely informational tool, balance isn't affected.

Related

Electronic Circuit — a crafting component (centre). Tin — tin ingots (the top row and the sides of the crafting). The /ala net command — the whole-dimension aggregated alternative (for operators).
